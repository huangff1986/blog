项目开发探索
============

着手一个项目的开始并不是直接上来就写代码。
应该先考虑清楚整个项目的大体思路后再进行开发。


# 需要考虑哪些技术点了？

1. 代码管理 git
2. 构建环境 webpack
3. 代码本身


## git

  由于只有我一个人开发，所以我的代码管理策略是由两个分支组成
  master 用于发布正式代码
  dev    用于实际的开发

  在dev中进行实际开发，当确保dev代码没问题后合并master分支，再将代码发生到git服务器上。

## webpack

  用于处理编译react代码。

## 代码

  使用react,redux,react-router。为核心代码进行开发
